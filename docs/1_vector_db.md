# Vector DB Options for Testing

간단하게 테스트 및 프로토타이핑에 적합한 Vector Database 옵션들을 정리합니다.

## 1. ChromaDB
Python 환경에서 가장 인기 있는 오픈소스 임베디드 Vector DB 중 하나입니다.

*   **특징**: 서버 설치 없이 Python 라이브러리로 바로 사용 가능. 로컬 파일 시스템에 데이터 저장.
*   **장점**:
    *   설치가 매우 간편함 (`pip install chromadb`).
    *   별도의 서버 프로세스 관리가 필요 없음 (Embedded 모드).
    *   LangChain, LlamaIndex 등 프레임워크와 연동이 잘 됨.
*   **단점**: 대규모 데이터셋에서는 성능 한계가 있을 수 있음.
*   **추천 상황**: 로컬 개발, 프로토타이핑, 복잡한 인프라 설정 없이 바로 시작하고 싶을 때.

## 2. FAISS (Facebook AI Similarity Search)
Meta에서 개발한 밀집 벡터의 효율적인 유사도 검색 및 클러스터링 라이브러리입니다.

*   **특징**: 메모리 내에서 매우 빠른 검색 속도 제공. DB라기보다는 라이브러리에 가까움.
*   **장점**:
    *   압도적인 검색 성능과 속도.
    *   GPU 가속 지원.
*   **단점**:
    *   데이터 영속성(Persistence) 관리를 직접 해야 함 (인덱스 파일 저장/로드).
    *   메타데이터 필터링 기능이 제한적임.
*   **추천 상황**: 극강의 검색 속도가 필요하거나, 커스텀 인덱싱 로직을 구현하고 싶을 때.

## 3. LanceDB
서버리스이며 임베디드 가능한 Vector DB로, Lance 데이터 포맷을 기반으로 합니다.

*   **특징**: Pandas/Arrow 에코시스템과 호환성이 좋음. 영속 저장소로 로컬 파일 또는 S3 등을 사용.
*   **장점**:
    *   설치가 간편함 (`pip install lancedb`).
    *   매우 가볍고 빠름.
    *   데이터 조작이 직관적임.
*   **단점**: 상대적으로 커뮤니티가 ChromaDB보다 작을 수 있음.
*   **추천 상황**: 데이터 분석 파이프라인과 통합하거나 매우 가벼운 솔루션을 원할 때.

## 4. Qdrant (Local Mode)
Qdrant는 원래 서버 클라이언트 구조이지만, 로컬 메모리 모드도 지원합니다.

*   **특징**: Rust로 작성되어 빠르고 안정적임. 로컬 모드 사용 시 서버 없이 사용 가능.
*   **장점**:
    *   강력한 메타데이터 필터링.
    *   추후 실제 프로덕션(Docker/Cloud)으로 마이그레이션하기 쉬움.
*   **단점**: 로컬 모드는 기능에 일부 제약이 있을 수 있음.
*   **추천 상황**: 나중에 실제 서버 배포를 염두에 두고 개발할 때.

## 결론 및 추천

**가장 추천하는 옵션: ChromaDB**

*   **이유**: `pip install` 만으로 바로 사용 가능하며, 별도의 설정 파일이나 도커 컨테이너 없이 파이썬 코드 내에서 `client = chromadb.PersistentClient(path="./db")` 형태로 즉시 영속성 있는 DB를 구성할 수 있어 초기 개발 및 테스트에 최적입니다.
